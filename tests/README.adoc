= Test suite

Integration tests live under `tests/rules/` and `tests/scenarios/`. Fixture data lives under `tests/fixtures/` with the same shape.

== Layout

* `tests/suite.zig` — imports test modules only (no test logic).
* `tests/harness.zig` — resolves fixture paths from rule/scenario ids.
* `tests/utils.zig` — shared diagnostic assertions.
* `tests/rules/{docs,complexity,style}/<rule_id>.zig` — one rule per file.
* `tests/scenarios/<name>.zig` — multi-rule or integration scenarios.
* `tests/fixtures/rules/{docs,complexity,style}/<rule_id>/{valid,invalid}/` — single-file cases.
* `tests/fixtures/rules/.../<rule_id>/project/{valid,invalid}/<case>/` — project roots (`root.zig`).
* `tests/fixtures/scenarios/<name>/` — scenario project trees.

== Adding a rule test

1. Add fixtures under `tests/fixtures/rules/docs/<rule_id>/valid|invalid/`.
2. Add `tests/rules/docs/<rule_id>.zig` with a `RuleLocator` constant and tests.
3. Import the file from `tests/suite.zig`.

No central fixture registry is required.

== CLI UX

Help rendering, unknown-flag parsing, and AsciiDoc docgen for the CLI are covered in the Fangz test suite (`modules/fangz/tests/cli_ux.zig`).
